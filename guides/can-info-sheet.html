Key Document Types

The golden rule: anything IN = ASN. Anything OUT = OOR.

ASN — Advanced Shipping Notification
Goods coming IN to any node. Created in SM²ARTA. WMS converts it to a Receiving Order.
OOR — Outbound Order Request
Goods going OUT of any node. Created in SM²ARTA. WMS converts it to a Shipping Order.
BOL — Bill of Lading
Created by Distribution Team. Follows each truck. Transport documentation for the carrier.
OS&D — Over, Short & Damage
Discrepancies or damaged items. Item stays at node but gets reclassified in SM²ARTA.
RDS — Room Data Sheet
Confirms movement from venue logistics compound to individual logical spaces (rooms, zones, FAs).

Section 08

SKU Codes & Nomenclature

Bob says
Suppliers all use different SKU formats. So we built our own FIFA nomenclature with 3 standardized segments. Once you learn the pattern, you can read any SKU at a glance!

General SKU Anatomy

Example: FN-CHA-002 = a Folding Chair

FN Item Category
-
CHA Item Type
-
002 Item Number

Categories: FN (Furniture), OS (Office Supply), FE (Fixture/Equip), etc.

Kit SKU Format

Kits bundle many individual items under one code for simpler logistics:

OSKIT Kit Type
-
001 Item Type
-
0XX FA Type
OSKIT = Office Supply Kit (~14 items, packed by Staples)
FNKIT = FF&E Kit (~12 items, tools/fixtures)
💡
Why kits? Instead of tracking 200 pens and staplers separately, we bundle into one kit SKU = one OOR line = one box. ML3K tracks 201 kits across 214 venues.

UoM (Unit of Measurement)

Each Box Kit Pack Pair

PPU (Pieces per UoM): Batteries = 10 per pack (PPU=10). Request 10 batteries → system converts to 1 pack automatically. ML3K handles all UoM math in the engine.

Section 11

Outbound Dispatch

Bob says
The big one — getting items out the door correctly. Each step exists for a reason!

1. Create OOR in SM²ARTA

Transaction #, destination, date/time, transport details, full item list with nomenclature + qty + UoM. WHS creates Shipping Order in WMS using OOR number.

🚫
No valid SM²ARTA transaction = dispatch held.

2. Picking & Staging

Pick per WMS order → Stage at loading area → Second person verifies codes, qty, UoM → Pack/wrap → 📸 Photo staged pallets. Report irregularities before loading.

3. Documentation

Prepare BOL (Bill of Lading) with vehicle #, driver name, seal #. All docs reviewed + signed before dispatch.

4. Vehicle Arrival & Loading

Record arrival → 📸 Photo plates + empty interior → Verify MHE → Load per plan (weight/stacking limits) → Straps + corner boards + wrap → Seal + record # → 📸 Photo loading + sealed door

5. Verify & Ship

Verify quantities in WMS → Upload all photos + docs → Record departure → Driver signs BOL → Confirm Shipping Order (auto-email sent)

6. SM²ARTA Confirmation

Enter WMS ref # → Verify date/qty/SKUs → Upload BOL + photos → Set status to "In Transit"

Section 12

SM²ARTA & ML3K

Bob says
Two systems, two jobs. SM²ARTA = asset tracking (what you have, where it is). ML3K = supply chain forecasting (what goes where, when, on which truck).

SM²ARTA

Asset Tracking & Transactions

  • Create ASNs, OORs, Moving transactions
  • Track every item from supplier to venue
  • Nomenclature management
  • Damage reports (OS&D)
  • 3 roles: Material Planner, Functional Planner, VLM

ML3K

Supply Chain Forecasting

  • LP Engine: plans 93+ truck loads to 8 destinations
  • LM Engine: plans per-venue truck loads from WHS
  • Late Engine: flags items that won't arrive on time
  • Assembly Timeline: tracks WHS processing dates
  • Kit management: 201 kits across 214 venues

Creating Nomenclature in SM²ARTA

Choose Type

LOG goods (~90%) or Functional goods. Mark if consumable, assembled, or branded.

Inherit or Create

Click "Inherit" to copy existing item data (UoM, price, source SKU), or enter manually + "Select Code" for new FIFA SKU.

Set UoM & PPU

Unit (Each, Box, Kit...) and pieces per unit. System auto-converts orders.

Asset Registration Form

For mid-tournament purchases that need tracking. Sent as Excel to all FAs and Venue Managers. Required fields:

Name of item
Quantity + UoM
Price
Package dimensions
Supplier SKU
Country of manufacture
HS Code (customs)

Section 13

Bob's Pro Tips

Bob's Final Words
Pin these to your wall. Tattoo them on your clipboard. Just don't forget them.

🚫 72-Hour Rule

Every OOR submitted 72+ hours before dispatch. No exceptions.

📸 When in Doubt, Photograph

Arrivals, departures, damage, seals, plates, staged cargo. Photos = insurance.

✅ Double Entry = Double Confirmation

Every transaction in both WMS and SM²ARTA. One system only = not done.

📄 No Document = No Movement

IN = ASN. OUT = OOR. Between rooms = RDS. Every physical movement has a matching document. Skip it and the warehouse holds your shipment.

📦 Track Everything — Even Surprises

Mid-tournament purchases? Asset Registration Form → Materials team → SM²ARTA.

🔧 ML3K + SM²ARTA Together

ML3K tells you what goes on which truck, when. SM²ARTA creates the OOR and tracks it. Use both!
